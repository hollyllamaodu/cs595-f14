	<h3 class="entry-title">
			Developing MAF Applications with OEPE
	    </h3>

    <h4 class="entry-meta">By Sydney-Oracle on <a href="#">Sep 09, 2014</a></h4>

    <div class="entry-body">
                                        	<div style="text-align: center;"> 
    <p><b><font face="georgia, times new roman, times, serif">Developing MAF Applications </font></b></p> 
    <p><b><font face="georgia, times new roman, times, serif">with</font></b></p> 
    <p><b><font face="georgia, times new roman, times, serif"></font></b><font face="georgia, times new roman, times, serif"><b>Oracle Enterprise Pack for Eclipse -&nbsp;</b></font><b><font face="georgia, times new roman, times, serif">OEPE</font></b></p> 
    <p style="text-align: left;"><font face="georgia, times new roman, times, serif">As previously mentioned in Blog Entry &quot;<a href="https://blogs.oracle.com/oepe/entry/android_and_ios_development_comes" title="Android and iOS development comes to OEPE 12.13.1">Android and iOS development comes to OEPE 12.13.1</a>&quot;, &nbsp;MAF development has&nbsp;certainly&nbsp;come to&nbsp;</font>Oracle Enterprise Pack for Eclipse (<span style="font-family: georgia, 'times new roman', times, serif;">OEPE). In many ways the development&nbsp;experience</span><span style="font-family: georgia, 'times new roman', times, serif;">&nbsp;is very smooth and it has a clean interface.</span></p> 
    <p style="text-align: left;">I can understand the development community preference and mass adoption of this IDE and for those of us who have become familiar with JDeveloper, here are some points to get you started and introduce you to the Eclipse development experience with MAF.</p> 
    <p style="text-align: left;"><b>Getting started ... OEPE Download</b></p> 
    <p style="text-align: left;">&nbsp;As with the JDeveloper installation, OEPE should be <a href="http://www.oracle.com/technetwork/developer-tools/eclipse/downloads/index.html" title="OEPE OTN Download Page">downloaded</a> and installed if a supported version of Eclipse is not already installed. I will use the &quot;Happy Path&quot; in this entry as the <a href="http://docs.oracle.com/cd/E50457_02/12131/OEPUG/install.htm" title="OEPE Installation Guide">documentation</a> does a great job in describing the various installation options.</p> 
    <p style="text-align: left;">As the download page states there are two versions of the OEPE but you should download your platform version &nbsp;of the &nbsp;12.1.3.1.1 with Oracle Mobile Application Framework (MAF). This distribution provides the traditional OEPE feature set with additional design time tools for mobile application development, the MAF runtime, and MAF sample applications. i.e. No need to update your installation afterwards with the MAF component extensions.</p> 
    <p style="text-align: left;"><b>Getting started ... OEPE SDK Configuration</b></p> 
    <p style="text-align: left;">&nbsp;Working with OEPE for MAF is similar to JDeveloper and as such you first steps will be to configure OEPE with the path to your Mobile SDKs, provide any keystore information for Android or provisioning profiles for iOS projects.</p> 
    <p style="text-align: left;">The process is similar to what we've been used, except it provides some smart wizards to get us through the process. From the Preferences menu option, drill down to Oracle -&gt; Mobile Application Framework and add you SDK settings.</p> 
    <p><b>IOS SDK Configuration</b></p> 
    <p> <img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_SDK_IOS_sm2.png" alt="MAF IOS SDK" align="middle" /></p> 
    <p><b>Android SDK Configuration</b></p> 
    <p><img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_SDK_Android_sm2.png" alt="MAF Android SDK settings" align="middle" /><br /></p> 
    <p> </p> 
    <p style="text-align: left;">Notice the <b>ADD</b>&nbsp;button on each screen which allows support of multiple SDKs, keystores, or provisioning profiles. </p> 
    <p> </p> 
    <p> </p> 
    <p style="text-align: left;"> <b>Getting Started ... Running a Sample ( O! I mean, an Example :D )</b></p> 
    <p style="text-align: left;">&nbsp;A good place to start is by verifying your configuration by deploying a MAF Sample Application. This is another small but nice difference in the OEPE user experience. To Open and Run a sample, simple create a new MAF Example. No need to find and then extract the samples to a project folder.</p> 
    <p style="text-align: left;">Create a New MAF Example Application, if this menu option is not showing yet, then just select Example and then you should find the MAF Example template selection item. Clicking next presents a selection of examples to choose from and I would suggest for your first MAF Workspace to select them all and have them conveniently located in a single location. &nbsp;<span style="text-align: center;">Once you've selected the example you may create a Run or Debug Configuration to deploy and test your application to a configured target.</span></p> 
    <p><b>New MAF Example</b></p> 
    <p><img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_New_Examples.png" alt="Create a New MAF Example Application" align="middle" /><br /></p> 
    <p><br /></p> 
    <p> </p> 
    <p style="text-align: left;">The first thing you will notice after the application wizard is finished is 3 sets of folders and not just 2. The wizard generates the standard projects we are used to; The Application, &amp; the ViewController or simply View projects. What is new is a new deployment project, or more correctly as per the documentation, the top-level assembly project.&nbsp;</p> 
    <p> </p> 
    <p><b>OEPE MAF Application Projects</b></p> 
    <p><img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_Example_Project_Folders.png" alt="OEPE MAF Project Folders" align="middle" /> </p> 
    <p> </p> 
    <p style="text-align: left;">&nbsp;The assembly project holds all the related packaging and deployment artifacts. &nbsp;Some of the files we have had to search for in the application resources view in JDeveloper, such as default images, application, logging or connections settings which lived in the various META-INF directories, now have a respectable consolidated area.</p> 
    <p style="text-align: left;">When you are ready to Run or Debug your mobile app then this is the project to choose to have it deployed to your test simulator, emulator, device or production package. To ensure that all is well with your environment I would suggested running one of the examples now.</p> 
    <p> </p> 
    <p style="text-align: left;"> </p> 
    <p style="text-align: left;"><b>Getting Started ... Deploying a MAF Application in OEPE</b> </p> 
    <p style="text-align: left;">&nbsp;In this process we don't talk about deploying an application but running one on a configured target configuration. To do this, select your top-assembly project (!Application &amp;&amp; !View) and select Run As or Debug As from the context menu. The configuration will defined the target to build and deploy the application.&nbsp;</p> 
    <p style="text-align: left;">Create a configuration for each of your target by select the assembly project, target platform, type of target with the required settings. The Advanced options are used to further refine the application settings found under the Jdeveloper Application properties -&gt; Deployment Profile -&gt; Target (Android || IOS) Profile -&gt; Target&nbsp;&nbsp;(Android || IOS)&nbsp;Options -&gt; Application Images.</p> 
    <p><b>Run Configuration for IOS</b></p> 
    <p><img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_Run_Configuration_sm.png" alt="IOS Run Configuration" /> </p> 
    <p> </p> 
    <p style="text-align: left;">Once defined, simply click run. </p> 
    <p style="text-align: left;">To run a configuration afterwards, simply select it from under the Debug or Run icons in the IDE.&nbsp;<img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_Debug_Run_icons.png" alt="Run and Debug Icons" /></p> 
    <p> </p> 
    <p style="text-align: left;">I'll cover configuring the Debugger later, if it proves problematic for the community but from my brief encounter, debugging is quite nice under eclipse and there is no need to open a secondary application to view the log file being created or using the Android Monitor to filter out your application messages.&nbsp;</p> 
    <p> </p> 
    <p style="text-align: left;"><b>Getting Started ... Creating a MAF Application in OEPE</b></p> 
    <p style="text-align: left;">So, what should you know about creating a MAF application in OEPE is that it is fairly similar to creating one in Jdeveloper, with the exception that you have the assembly project. You have different perspectives to perform the various tasks based on the type of component you are working on. You have the same relationship between the Application and the View projects with some small differences; such as the new MAF folder which houses the Data Control Manager, the editor interface for the DataControl.dcx file.</p> 
    <p><img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_Assembly_Proj_MAF_folder_sm.png" alt="MAF Assembly Project" /> <img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_Projects_MAF_folder_sm2.png" alt="MAF Application &amp; View Projects" /></p> 
    <p> </p> 
    <p style="text-align: left;">In the assembly project the MAF folder displays the MAF Application Editor, the editor for the maf-application.xml. This project also exposes the application resource and other configuration files, normally in the resources view in JDeveloper and the hidden .adf folder of the application.</p> 
    <p style="text-align: left;">After the creation wizard is complete you will have everything you need to start developing your MAF application, except the connections.xml file which is used to keep track of your application resource links. The easiest way to create this file is from the MAF Application Editor. Once the editor is open, select the security component or tab, for lack of a better term, and you will find a hyper link for the connections.xml file which is also used to store URL White Lists. Simply select the link and you will be prompted to create the connections.xml.</p> 
    <p><b>Create Connections.xml&nbsp;</b></p> 
    <p><img src="https://blogs.oracle.com/mobile/resource/OEPE4MAF/MAF_Application_Editor_sm2.png" alt="Create Connections.xml from the MAF Application Editor" align="middle" /> </p> 
    <p> </p> 
    <p style="text-align: left;">Getting this created was the easy part and for &nbsp;SOAP based resources this ease of use continues but for those focused on the consumption of Restful services, you will have to create the connection entry manually. I will provide a sample connect entry at the end for those of you who do not have access to Jdeveloper and the Rest Data Control wizard. </p> 
    <p style="text-align: left;">This feature will be released in a future version of OEPE for MAF but once the connection URL has been added, consuming, transforming and exposing the Restful services and data is the same process and APIs.<span style="text-align: right;"> </span></p> 
    <p style="text-align: left;">Some other differences to mention are:</p> 
    <p> </p> 
    <ul> 
      <li style="text-align: left;">The Palette area combines the Component and Data Control perspectives in Jdeveloper</li> 
      <li style="text-align: left;">The Structure pane is called Outline, which shows the xml document format and not the component type icons, which make it easier to develop pages/views as in JDeverloper. You can still edit your page/view in this manner but be careful to read the component text and its level before dropping your new component onto the page.</li> 
      <li style="text-align: left;">Flow actions, defined in your task flow, are only listed in the &quot;Select a Value&quot; / &quot;Select an Outcome&quot; screen and not in the drop down list of the action property. The pulldown only list &quot;__back&quot;, the default return action</li> 
      <li style="text-align: left;">The Console window has multiple views and you will need to click / select it to switch between the build/deployment view and the running application log view</li> 
      <li style="text-align: left;">To debug your application you need to configure a Remote Java Application configuration to connect to the Debug port connection property</li> 
      <li style="text-align: left;">To Debug your application, start the application first then the debugger</li> 
    </ul> 
    <p> </p> 
    <p> </p> 
    <p style="text-align: center;"> <b>Summary</b></p> 
    <p style="text-align: left;">&nbsp;To wrap up, I'd just like to say that OEPE development experience for MAF is good and only lacks a few wizards found in JDeveloper. It supports multiple SDK environments and versions as well as keystone and provisioning profiles, which will come in handy for customer development projects.</p> 
    <p style="text-align: left;">The integrated log console view is great as well as the debugger for the application's java code. The only negative exception is the lack of wizard to add Rest URL connections, which I hope we will see released soon. For a first release this is great start and I look forward to seeing what the future holds for both OEPE and MAF development.<span style="text-align: center;"> </span></p> 
    <p style="text-align: left;">Please add your experiences and comments, so the PMs can keep improving OEPE and MAF in general.<span style="text-align: center;"> </span></p> 
    <p style="text-align: left;">Regards Sydney&nbsp;</p>
